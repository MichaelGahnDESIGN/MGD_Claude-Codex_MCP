# SKILLS

Dieser Ordner ist die zentrale Sammelstelle für wiederverwendbare Skills.

## Ziel

`VORLAGE/AI/SKILLS/` soll so aufgebaut sein, dass später jederzeit neue Themenbereiche, Aufgabenarten und Unterordner sauber hinzugefügt werden können.

## Strukturregel

Im Root von `VORLAGE/AI/SKILLS/` liegt nur diese Datei als Einstieg.

Die eigentlichen Inhalte werden in Unterordnern organisiert, zum Beispiel:

- `Allgemein/`
- `Content/`
- `WordPress/`
- `Shopsysteme/`

Jeder Unterordner soll nach Möglichkeit eine eigene `SKILL.md` enthalten, die kurz erklärt:

- wofür der Bereich gedacht ist
- welche Arten von Skills dort hineingehören
- welche Benennungslogik genutzt wird
- wann ein neuer Unterordner sinnvoller ist als ein weiterer Skill in diesem Bereich

## Flexible Unterordner

Unterordner dürfen je nach Bedarf wachsen.

Erlaubt und sinnvoll sind zum Beispiel:

- nur einzelne Skill-Dateien
- zusätzliche Beispiele
- Vorlagen
- Referenzen
- spätere weitere Unterordner

Es gibt bewusst keine starre Pflichtstruktur pro Unterordner.

## Benennungsregeln

- Dateinamen sollen klar und zweckorientiert sein.
- Ein Skill soll möglichst genau eine Aufgabe oder ein klares Aufgabengebiet abdecken.
- Wenn ein Bereich zu breit wird, soll ein neuer Unterordner entstehen statt einer unübersichtlichen Dateiliste.
- Allgemeine, projektübergreifende Skills gehören eher nach `Allgemein/`.
- Plattform- oder systemspezifische Skills gehören in ihren jeweiligen Fachordner.
