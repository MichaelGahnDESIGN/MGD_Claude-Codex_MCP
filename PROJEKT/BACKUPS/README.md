# BACKUPS

Dieser Ordner enthält Projektsicherungen, Exportstände und Wiederherstellungsgrundlagen.

Backups bleiben lokal und werden standardmäßig nicht im Root-Repository
versioniert. Die README bleibt als Orientierung erhalten.

Empfohlene Benennung:

- `YYYY-MM-DD_HH-mm-ss_vVERSION`
- oder `YYYY-MM-DD_HH-mm-ss_vVERSION.zip`

Zu jedem relevanten Backup soll in `PROJEKT/DOKUMENTATION/Projektbetrieb/Backups.md` dokumentiert werden:

- was gesichert wurde
- wann gesichert wurde
- warum gesichert wurde
- wie die Wiederherstellung erfolgt

Aktueller Hinweis:

- `dokumentation-git-backup-2026-06-07.git/` enthält die frühere innere
  Git-Historie von `PROJEKT/DOKUMENTATION/`. Sie wurde gesichert, damit die
  Vorlage als ein aufgeräumtes Root-Git-Repository funktionieren kann.
